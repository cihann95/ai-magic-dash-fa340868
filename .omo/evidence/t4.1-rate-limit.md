## Implementation Details

### Algorithm
- **Sliding Window**: Uses Redis sorted sets with timestamps as scores
- **Key Pattern**: `ratelimit:{user_id}:{route}`
- **Atomicity**: Redis operations are sequential (zremrangebyscore → zcard → zadd → expire)
- **Fail-Open**: If Redis is unavailable, requests are allowed through

### Response Format (429)
```json
{
  "error": "Rate limit exceeded",
  "retry_after": 42,
  "limit": 20,
  "remaining": 0
}
```

### Response Headers
- `X-RateLimit-Limit`: Maximum requests per window
- `X-RateLimit-Remaining`: Remaining requests in current window
- `X-RateLimit-Reset`: Unix timestamp when window resets
- `Retry-After`: Seconds until rate limit resets
