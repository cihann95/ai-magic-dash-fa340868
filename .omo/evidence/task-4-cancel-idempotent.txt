# Task 4 QA: Cancel Idempotent (Kuyrukta Değilse)

## Test Senaryosu
Kullanıcı kuyrukta DEĞİL (real_balance_locked=0), cancel çağırır.

## Beklenen Davranış
1. Cancel handler çalışır
2. Redis lrem 0 döner (kuyrukta yok)
3. DB conditional UPDATE: `WHERE real_balance_locked >= entry_fee` koşulu sağlanmaz (0 >= 5 false)
4. UPDATE 0 row affected, hata VERMEZ
5. 200 OK döner: `{ok: true}`

## Test Adımları (curl)
