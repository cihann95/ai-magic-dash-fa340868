# T0.4 — Activity Feed Tablosu / View Güvenlik Analizi

## View Tanımı

### İlk Oluşturma (20260421082651)
```sql
CREATE OR REPLACE VIEW public.activity_feed AS
SELECT ... FROM public.trades t
JOIN public.public_profiles pp ON pp.user_id = t.user_id
WHERE pp.is_active = true AND pp.show_trades = true
UNION ALL
SELECT ... FROM public.user_achievements ua
JOIN public.public_profiles pp ON pp.user_id = ua.user_id
WHERE pp.is_active = true;
GRANT SELECT ON public.activity_feed TO authenticated;
```
**Sorun**: SECURITY DEFINER yoktu → view owner (postgres)rolleriyle çalışıyordu → RLS bypass riski.

### Güncelleme (20260506090228)
```sql
DROP VIEW IF EXISTS public.activity_feed;
CREATE VIEW public.activity_feed
WITH (security_invoker = true) AS
SELECT ... FROM public.trades t
JOIN public.public_profiles pp ON pp.user_id = t.user_id
WHERE pp.is_active = true AND pp.show_trades = true
UNION ALL
SELECT ... FROM public.user_achievements ua
JOIN public.public_profiles pp ON pp.user_id = ua.user_id
WHERE pp.is_active = true AND pp.show_portfolio = true;
```
**Düzeltme**: `security_invoker = true` eklendi → view sorgulayan kullanıcının rollerini kullanıyor → RLS politikaları uygulanıyor.

### Son Güncelleme (20260421082708)
```sql
ALTER VIEW public.activity_feed SET (security_invoker = on);
```
**Onay**: `security_invoker` tekrar açıkça ayarlandı.

## Güvenlik Analizi

| Özellik | Durum | Açıklama |
|---|---|---|
| SECURITY DEFINER | ❌ YOK | View owner rolleriyle çalışmıyor |
| security_invoker | ✅ TRUE | Sorgulayan kullanıcının rolleri kullanılıyor |
| RLS Bypass | ❌ YOK | Base tabloların RLS politikaları uygulanıyor |
| SELECT Grant | ✅ authenticated | Sadece giriş yapmış kullanıcılar erişebilir |

## Base Tabloların RLS Durumu
- `trades`: RLS enabled, INSERT/SELECT/UPDATE/UPDATE policies (owner-based)
- `public_profiles`: RLS enabled, SELECT (public), INSERT/UPDATE (owner)
- `user_achievements`: RLS enabled, SELECT (public), INSERT (owner via trigger)

## Sonuç
- **View güvenli**: `security_invoker = true` sayesinde RLS bypass etmiyor
- **Veri sızıntısı riski DÜŞÜK**: View sadece `is_active = true` olan profillerin trade/achievement'lerini gösteriyor
- **Potansiyel sorun**: `show_trades = true` ve `show_portfolio = true` filtreleri kullanıcı kontrolünde — hassas veri gösterimi kullanıcıya bağlı
- **Öneri**: View'ın exposure'unu daraltmak için `show_trades`/`show_portfolio` default'unu `false` yap

## Durum: PASS (conditionally)
- View şu an güvenli
- `security_invoker = true` doğru ayarlanmış
- Ancak `show_trades`/`show_portfolio` default değerleri kontrol edilmeli
