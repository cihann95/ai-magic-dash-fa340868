## Açık 1: execute-trade Race Condition — KAPATILDI

### Sorun
`supabase/functions/execute-trade/index.ts` satır 142-184: Position close işleminde optimistic lock yoktu. İki eş zamanlı `position_id` ile kapatma isteği ikisi de başarılı oluyordu → bakiye çift kredi alınıyordu.

### Fix
- `positions` tablosuna `closed_at` kolonu eklendi (migration `20260610000006`)
- Position okunduktan sonra `closed_at` null mu kontrol ediliyor
- Atomik `update({ closed_at: now() }).is("closed_at", null)` ile optimistic lock uygulanıyor
- Update 0 satır etkilerse → race condition hatası döndürülüyor
- Ancak başarırsa → silme işlemine devam ediliyor

### Dosyalar
- `supabase/functions/execute-trade/index.ts` — lines 142-196
- `supabase/migrations/20260610000006_race_condition_and_invariant.sql` — positions.closed_at + index

## Açık 2: settlement_ledger Invariant — KAPATILDI

### Sorun
`settlement_ledger` tablosunda `pot_total = prize_amount + fee_collected` ilişkisi zorlanmıyordu. Mevcut CHECK constraint'leri sadece `>= 0` kontrolü yapıyordu.

### Fix
- `validate_settlement_invariant()` trigger fonksiyonu oluşturuldu
- `settlement_invariant_check` BEFORE INSERT OR UPDATE trigger'ı eklendi
- Status `completed` olduğunda `pot_total = prize_amount + fee_collected` zorlanıyor
- Invariant ihlali `RAISE EXCEPTION` ile reddediliyor

### Dosyalar
- `supabase/migrations/20260610000006_race_condition_and_invariant.sql` — trigger + fonksiyon
