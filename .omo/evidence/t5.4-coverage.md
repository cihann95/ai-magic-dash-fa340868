## 3. Coverage Analysis

### 3.1 Why Coverage is Low (10.13%)

The overall coverage metric is low because the coverage report includes **all files in the project**, including:

1. **Edge function source files** (19 files, 0% coverage) — tested in separate vitest instance with different config
2. **Type definition files** (`src/types/*.ts`, `supabase/functions/_shared/*.ts`) — 0% coverage
3. **UI components** (`src/components/ui/*.tsx`) — shadcn/ui components, not tested directly
4. **Pages** — Only 4/20+ pages have tests (Settings, Blitz, Index, Portfolio)
5. **Hooks** — Only 3/15+ hooks have tests
